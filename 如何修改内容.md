# 如何修改内容

这是一套网页式 PPT。你不需要先理解整个项目，先记住一个入口：

**主要改内容：`src/slides.tsx`**

## 最建议新手改哪里

只改 `src/slides.tsx`。

你可以在这个文件里改：

- 每页标题
- 每页正文
- 卡片文案
- 图表里的示例数据
- Agenda 和左侧 hover 导航显示的页名
- 增加、删除或调整页面顺序

当前内容必须围绕六幕叙事：

1. 问题
2. 转折
3. 案例
4. 系统化
5. 升级
6. 结论

不要新增只为展示组件而存在的页面。每一页都要有明确观点，并服务“AI Agent 工作流发布会：从工具调用到自动化协作系统”这个主题。

## 不建议新手先动哪里

- `src/deck/Deck.tsx`：这是翻页、全屏、进度条、目录等结构。
- `src/deck/controls/`：这是按钮、hover 目录、进度条。
- `src/deck/charts/`：这是自绘图表组件。
- `src/deck/media/MediaPreview.tsx`：这是图片 lightbox、视频占位和媒体弹层组件。
- `src/styles.css`：这是整体视觉、动态背景、响应式和打印样式。只有想改风格时再动。

## 改标题在哪里

打开 `src/slides.tsx`，找到 `slideMeta`，里面每一项都有：

```ts
title: '这一页的大标题'
navTitle: '目录里的短标题'
section: '所属章节'
```

## 改每页正文在哪里

还是在 `src/slides.tsx`。每一页都有一个 `content:`，里面就是这一页看到的正文、卡片、流程和图表。

## 改按钮文案在哪里

翻页、全屏等按钮在：

```text
src/deck/controls/TopControls.tsx
```

左侧 Agenda 的固定控件在：

```text
src/deck/controls/HoverAgenda.tsx
```

不要把可见文案改成粗糙的单字，例如“钉”“锁”。小按钮里也不要放 `PINNED` 这种长词；建议使用 `PIN` / `ON` 这类短词，完整含义放在 tooltip / title。

## 改页数怎么做

在 `src/slides.tsx` 里新增或删除 `slides` 数组中的一页，同时保持 `slideMeta` 对应。Agenda、hover 导航、页码和进度条会自动跟着变化。

## 改颜色和整体风格在哪里

主要在：

```text
src/styles.css
```

常用变量在文件开头的 `:root` 里。默认主题优先改语义 token，例如：

```css
--color-accent-primary
--color-accent-secondary
--color-accent-tertiary
--color-accent-warm
--color-accent-success
--color-accent-risk
--chart-color-1 ... --chart-color-6
```

默认视觉基线是 `Original Agent Skill Lab Palette / agent-lab`，色彩母版来自早期本地项目 `/Users/water/Documents/系统操作/agent-skill-html-deck`。它只参考旧项目的色彩和 hover 质感：deep navy / blue black 背景、左上 cyan / teal 光、右侧 muted violet / indigo 空间光、深蓝黑玻璃舞台、cyan / teal Agenda active state、cyan / mint / violet 进度条。改风格时先保护默认链接，再考虑实验增强版。

本轮默认版做过一次 palette reconstruction：不要把默认主题做成黑金、沉闷冷灰或随机 neon。允许在主题边界内微调 palette、gradient、surface、hover，但默认 tone 应继续接近 `agent-skill-html-deck` 的原始高级深色科技感，并服务“AI Agent 工作流发布会 / 从工具调用到自动化协作系统”。

默认主题的 foreground 要克制：主标题使用主文字色，subtitle / eyebrow / section label 使用统一的 quiet token，caption / chart note / legend 使用更弱的 caption token。不要让普通说明文字在 cyan、blue、violet、amber、coral 之间随机跳色。

底色和交互优先改这些语义 token，而不是到处写死颜色：

```css
--surface-0
--surface-1
--surface-2
--surface-3
--surface-border
--surface-glow
--hover-surface
--hover-border
--hover-glow
--hover-lift
```

舞台空间优先改这些布局 token，而不是用整体缩放：

```css
--app-safe-x
--app-safe-y
--topbar-height
--topbar-to-stage-gap
--stage-bottom-gap
--progress-height
--stage-radius
--agenda-width
--stage-left
--stage-right
--stage-top
--stage-bottom
```

当前默认版已把外层背景压回更深的 blue-black，并把 topbar 到舞台的距离收紧。继续优化时先保护 `1366x768` 和 `1440x900` 的 16:9 内容显示区，不要用 `scale()` / `zoom` 把整套 slide 压小。

## 动态背景在哪里调

在 `src/styles.css` 顶部调这些变量：

```css
--bg-motion-opacity
--bg-glow-opacity
--bg-sweep-opacity
--bg-particle-opacity
```

如果文字开始不清楚，先降低这些变量，不要降低正文透明度。`champagne` / `visual` / `aurora` 只能作为实验增强版，不要把它们的强背景效果、黑金色调或极光强度带回默认主题。默认主题也不要被过度修正成冷灰单调，默认配色应继续贴近旧项目母版。

默认背景应该使用柔和的多层 radial / conic / linear gradients、超大 halo、斜向 ribbon、低透明网格和 aurora glow 建立空间深度，不能退回单一黑灰底。光晕层和模糊层要覆盖到视口外，避免顶部、底部或左右出现黑色硬边。

## 图表入场动效在哪里调

图表组件在：

```text
src/deck/charts/
```

进度条在：

```text
src/deck/controls/Progress.tsx
```

这些组件会在进入当前 slide 时重新挂载并播放入场动效。CSS 动画时长和 easing 主要在 `src/styles.css`，变量是 `--chart-ease`。

## 长内容卡片怎么改

长内容卡片使用：

```tsx
<ScrollableCard label="QA 02" title="长内容卡片">
  ...
</ScrollableCard>
```

组件定义在：

```text
src/deck/layouts/SlideBlocks.tsx
```

样式在 `src/styles.css` 的 `.scrollable-card` 和 `.scrollable-card-body`。原则是：页面、stage 和 slide frame 不滚动，只让过长卡片内部滚动。适合验收清单、长列表、代码片段或资料摘录；如果内容可以拆页，优先拆页。

## 图片和视频怎么改

媒体组件在：

```text
src/deck/media/MediaPreview.tsx
```

当前 Deck 能力页在 `src/slides.tsx` 里演示：

```tsx
<MediaImage title="Image lightbox" caption="..." alt="...">
  ...
</MediaImage>

<MediaVideo title="Video preview" caption="..." />
```

`MediaImage` 适合截图、设计稿、架构图和图表预览。默认点击后打开 lightbox，放大层会 contain 在视口内，并支持 overlay、Close 和 `Esc` 关闭。

`MediaVideo` 当前是可替换的本地视频占位示例，展示播放、暂停、进度、音量和放大弹层的交互。接入真实视频资产时，继续保持 16:9 aspect-ratio、默认不 autoplay、关闭弹层或切换 slide 时暂停。

媒体弹层打开时会暂时阻止左右方向键翻页，避免用户查看图片或视频时误切 slide。关闭弹层后，键盘翻页恢复。

## 布局和装饰标签注意事项

- hover 导航是浮层，不应该推动主舞台；只有 pin 固定后才允许舞台缩进。
- Pinned Agenda 是双栏 reflow，不是整张 slide 缩放。固定目录后通过网格、卡片和图表重新排布适配剩余舞台宽度，不要用 `scale()` / `zoom` 把幻灯片整体压小。
- topbar、Agenda、stage 和 progress 共用 layout token；改垂直空间时优先调 token，不要在多个组件里散落硬编码 offset。
- 装饰性英文标签要放在顶部 eyebrow、左上角或底部安全区，不能压住正文、图表和卡片。
- split layout 要看视觉重量平衡，一侧文字时，另一侧最好有图表、流程、节点图或视觉锚点。
- Agenda panel 要保持统一内边距，第一项不能被标题区或边框裁切。
- 小 Pin 控件不能显示超过 3 个英文字母的长词。
- 主标题宽度应默认匹配主体内容容器，不要用过窄 max-width 造成过早折行。
- 图表页只讲一个核心数据观点，避免把多个组件像 dashboard 一样拼在一起。
- 主要卡片、图表卡、Agenda item、按钮、流程节点和架构节点要共享统一 hover 语言：轻微上浮、surface 提亮、边框增强、低透明 glow、accent 轻微变亮，并支持 reduced motion。
- 单张卡片内容过长时，只允许卡片内部滚动，不允许页面级滚动；滚动区域要有明确高度、细滚动条和可聚焦状态。
- 图片 / 视频媒体必须放在稳定 aspect-ratio frame 内，弹层 z-index 要高于 Agenda、topbar 和 progress，且不能影响 16:9 舞台布局。
- Design QA 不只看 token 是否存在，也要看实际观感是否像高级 AI 产品发布会。当前项目已参考 `ui-ux-pro-max` 和 `frontend-design` 做 companion design review。
