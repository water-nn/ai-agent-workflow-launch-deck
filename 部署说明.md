# 部署说明

## 本地预览

macOS：双击 `打开预览.command`。

Windows：双击 `打开预览.cmd`。

打开后会出现一个黑色命令窗口，请不要关闭。它负责启动本地网页服务，并会自动打开浏览器。

## 修改内容

主要修改：

```text
src/slides.tsx
```

改完后可以重新双击预览文件查看效果。

## 重新构建

```bash
npm run build
```

## GitHub Pages

本项目的 GitHub Pages 目标链接是：

```text
https://water-nn.github.io/ai-agent-workflow-launch-deck/
```

可视化主题链接：

```text
https://water-nn.github.io/ai-agent-workflow-launch-deck/?theme=visual
```

石墨极光色调链接：

```text
https://water-nn.github.io/ai-agent-workflow-launch-deck/?tone=aurora
```

可视化主题 + 石墨极光链接：

```text
https://water-nn.github.io/ai-agent-workflow-launch-deck/?theme=visual&tone=aurora
```

Vite 项目部署到 GitHub Pages 项目仓库时，`vite.config.ts` 必须包含：

```ts
base: '/ai-agent-workflow-launch-deck/'
```

## 如何更新远程网页

改完内容后执行：

```bash
npm run build
git add .
git commit -m "Update presentation content"
git push
```

push 后 GitHub Actions 会自动重新构建并发布 GitHub Pages。

## 发布后检查

发布后建议检查：

- 默认主题是否正常。
- `?theme=visual` 是否正常。
- `?tone=aurora` 是否正常。
- `?theme=visual&tone=aurora` 是否正常。
- hover 导航展开时主舞台不跳动。
- pin 固定后主舞台轻微缩进并重新居中。
- 装饰性英文标签不遮挡正文、图表或卡片。
- 图表页有入场动效，且 reduced motion 下动效会降低或关闭。
- 页面级无滚动，键盘上下左右可翻页。

## 导出 PDF

1. 本地打开预览。
2. 在浏览器中进入需要打印的页面。
3. 按 `Command + P` 或 `Ctrl + P`。
4. 目标选择“保存为 PDF”。
5. 纸张方向选择横向，尽量关闭页眉页脚。

## 常见问题

如果远程页面空白，优先检查 `vite.config.ts` 的 `base` 是否是：

```ts
base: '/ai-agent-workflow-launch-deck/'
```

如果 GitHub Actions 失败，请打开 GitHub 仓库页面的 **Actions** 标签查看报错。

## 设计维护说明

hover 导航默认作为浮层覆盖，是为了避免鼠标经过目录时主舞台跳动。只有用户明确点击 Pin 固定目录时，舞台才会重新计算空间。

图表页应该像演示文档，不要像 dashboard 拼贴。每页只讲一个核心观点，图表标题尽量写成结论句。

多色调使用 CSS token 管理。当前已实现 `champagne` 和 `aurora`；后续扩展新 tone 时，应同时调整按钮、导航、卡片、图表、背景和进度条，而不是只换背景色。
