# 部署说明

## 本地预览

macOS：双击 `打开预览.command`。

Windows：双击 `打开预览.cmd`。

打开后会出现一个黑色命令窗口，请不要关闭。它负责启动本地网页服务，并会自动打开浏览器。

## 修改内容

主要修改：

```text
src/slides.tsx
```

改完后可以重新双击预览文件查看效果。

## 重新构建

```bash
npm run build
```

## GitHub Pages

本项目的 GitHub Pages 目标链接是：

```text
https://water-nn.github.io/ai-agent-workflow-launch-deck/
```

Vite 项目部署到 GitHub Pages 项目仓库时，`vite.config.ts` 必须包含：

```ts
base: '/ai-agent-workflow-launch-deck/'
```

## 如何更新远程网页

改完内容后执行：

```bash
npm run build
git add .
git commit -m "Update presentation content"
git push
```

push 后 GitHub Actions 会自动重新构建并发布 GitHub Pages。

## 导出 PDF

1. 本地打开预览。
2. 在浏览器中进入需要打印的页面。
3. 按 `Command + P` 或 `Ctrl + P`。
4. 目标选择“保存为 PDF”。
5. 纸张方向选择横向，尽量关闭页眉页脚。

## 常见问题

如果远程页面空白，优先检查 `vite.config.ts` 的 `base` 是否是：

```ts
base: '/ai-agent-workflow-launch-deck/'
```

如果 GitHub Actions 失败，请打开 GitHub 仓库页面的 **Actions** 标签查看报错。
