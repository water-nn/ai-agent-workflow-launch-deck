# 部署说明

## 本地预览

macOS：双击 `打开预览.command`。

Windows：双击 `打开预览.cmd`。

打开后会出现一个黑色命令窗口，请不要关闭。它负责启动本地网页服务，并会自动打开浏览器。

## 修改内容

主要修改：

```text
src/slides.tsx
```

改完后可以重新双击预览文件查看效果。

## 重新构建

```bash
npm run build
```

## 如何更新远程网页

改完内容后执行：

```bash
npm run build
git add .
git commit -m "Update presentation content"
git push
```

push 后 GitHub Actions 会自动重新构建并发布 GitHub Pages。

## 发布后检查

发布后建议检查：

- 默认主题是否保持高级深色科技发布会风格。
- 默认主题是否不再是黑金 / 香槟金主导。
- 默认主题是否不再是沉闷冷灰模板，是否有受控的 aqua cyan、deep electric blue、muted indigo、teal、amber、soft coral 图表 palette。
- 默认背景是否有多层渐变空间感，而不是平面黑灰底。
- 顶部、底部和左右边缘是否没有黑色硬边、mask 硬线或 blur 接缝。
- 顶部 controls 区域是否与背景融合，没有横向黑色轮廓。
- surface 层级是否清晰：body、stage、card、navigation、chart panel、hover / active 状态不要混成一个灰色。
- subtitle、eyebrow、caption、chart note、legend 是否使用统一 quiet token，而不是随机彩色文字。
- 卡片、按钮、Agenda item、图表卡、流程节点和架构节点 hover 是否有明显但克制的高级反馈，不影响布局，也没有强烈廉价发光。
- 默认主题是否仍围绕 Agent Skill、html-presentation-deck、GitHub 交接和 Design QA 叙事。
- `?tone=champagne` 是否正常，并且没有反向污染默认主题。
- `?theme=visual` 是否正常，并且没有反向污染默认主题。
- `?tone=aurora` 是否正常，并且没有反向污染默认主题。
- `?theme=visual&tone=aurora` 是否正常，并且没有反向污染默认主题。
- hover 导航展开时主舞台不跳动。
- pin 固定后进入双栏 reflow，主舞台使用剩余空间重新排布，不把整张 slide 缩小成小图。
- 装饰性英文标签不遮挡正文、图表或卡片。
- 图表页有入场动效，且 reduced motion 下动效会降低或关闭。
- 页面级无滚动，键盘上下左右可翻页。

## 导出 PDF

1. 本地打开预览。
2. 在浏览器中进入需要打印的页面。
3. 按 `Command + P` 或 `Ctrl + P`。
4. 目标选择“保存为 PDF”。
5. 纸张方向选择横向，尽量关闭页眉页脚。

## 常见问题

如果远程页面空白，优先检查 `vite.config.ts` 的 `base` 是否是：

```ts
base: '/ai-agent-workflow-launch-deck/'
```

如果 GitHub Actions 失败，请打开 GitHub 仓库页面的 **Actions** 标签查看报错。

## 设计维护说明

hover 导航默认作为浮层覆盖，是为了避免鼠标经过目录时主舞台跳动。只有用户明确点击 Pin 固定目录时，舞台才会重新计算空间。Pinned Agenda 必须是 responsive reflow，不允许通过整体 `scale()` / `zoom` 把幻灯片压小。

图表页应该像演示文档，不要像 dashboard 拼贴。每页只讲一个核心观点，图表标题尽量写成结论句。

多色调使用 CSS token 管理。当前已实现默认 `slate` 的 Expressive Premium Dark Tech palette、可选 `champagne` 和实验 `aurora`；后续扩展新 tone 时，应同时调整按钮、导航、卡片、图表、背景、hover 状态和进度条，而不是只换背景色。

本轮视觉重设计已参考 `ui-ux-pro-max` 和 `frontend-design` 做 Design QA。后续发布检查不应只验证 token，还要人工判断实际画面是否像高级 AI 产品发布会。

跨设备继续项目时，先读 `PROJECT_STATUS.md`。它记录当前主题、稳定链接、默认视觉基线和下一步建议。
