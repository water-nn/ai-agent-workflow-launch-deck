## 发布后检查

发布后建议检查：

- 默认主题是否保持高级深色科技发布会风格。
- 默认无参数链接是否等价于 `?tone=agent-lab`。
- 默认背景是否明显接近旧项目的 deep navy / blue black + 左 cyan / 右 violet 空间光。
- 外层背景是否更深、更有旧项目母版的空间感，但不是纯黑或平面黑灰。
- `1366x768` 和 `1440x900` 下 topbar 到舞台的距离是否足够紧凑，16:9 内容显示区是否明显更高。
- stage spacing 是否通过 `--app-safe-x`、`--app-safe-y`、`--topbar-height`、`--topbar-to-stage-gap`、`--stage-bottom-gap`、`--stage-left`、`--stage-right` 等 layout token 管理，而不是散落硬编码。
- 默认主题是否不再是黑金 / 香槟金主导。
- 默认主题是否不再是沉闷冷灰模板，是否有受控的 cyan、teal、electric blue、muted violet、indigo、mint 图表 palette。
- 默认背景是否有多层渐变空间感，而不是平面黑灰底。
- 顶部、底部和左右边缘是否没有黑色硬边、mask 硬线或 blur 接缝。
- 顶部 controls 区域是否与背景融合，没有横向黑色轮廓。
- surface 层级是否清晰：body、stage、card、navigation、chart panel、hover / active 状态不要混成一个灰色。
- subtitle、eyebrow、caption、chart note、legend 是否使用统一 quiet token，而不是随机彩色文字。
- 卡片、按钮、Agenda item、图表卡、流程节点和架构节点 hover 是否有明显但克制的高级反馈，不影响布局，也没有强烈廉价发光。
- 默认主题是否仍围绕 Agent Skill、html-presentation-deck、GitHub 交接和 Design QA 叙事。
- `?tone=champagne` 是否正常，并且没有反向污染默认主题。
- `?theme=visual` 是否正常，并且没有反向污染默认主题。
- `?tone=aurora` 是否正常，并且没有反向污染默认主题。
- `?theme=visual&tone=aurora` 是否正常，并且没有反向污染默认主题。
- hover 导航展开时主舞台不跳动。
- pin 固定后进入双栏 reflow，主舞台使用剩余空间重新排布，不把整张 slide 缩小成小图。
- Design QA 页的长内容卡片是否只在卡片内部滚动，页面本身是否仍无滚动。
- 长内容卡片滚动条、focus 状态和 hover 反馈是否统一且不跳布局。
- Deck 能力页的图片预览是否保持 aspect-ratio，点击后 lightbox 是否高于 Agenda、topbar 和 progress。
- 图片 lightbox 是否支持 overlay、Close 和 `Esc` 关闭，关闭后方向键翻页恢复。
- 视频占位是否有播放、暂停、进度、音量和放大控制，默认不自动播放。
- 视频放大弹层关闭后是否暂停，媒体弹层打开时方向键是否不会误切 slide。
- 装饰性英文标签不遮挡正文、图表或卡片。
- 图表页有入场动效，且 reduced motion 下动效会降低或关闭。
- 页面级无滚动，键盘上下左右可翻页。

## 设计维护说明

hover 导航默认作为浮层覆盖，是为了避免鼠标经过目录时主舞台跳动。只有用户明确点击 Pin 固定目录时，舞台才会重新计算空间。Pinned Agenda 必须是 responsive reflow，不允许通过整体 `scale()` / `zoom` 把幻灯片压小。

舞台高度优化优先调整 `src/styles.css` 顶部 layout token。不要用整体缩放、固定 1920x1080 画布或压小正文来解决上方留白过大的问题。

图表页应该像演示文档，不要像 dashboard 拼贴。每页只讲一个核心观点，图表标题尽量写成结论句。

多色调使用 CSS token 管理。当前已实现默认 `agent-lab` 的 Original Agent Skill Lab Palette、可选 `champagne` 和实验 `aurora`；后续扩展新 tone 时，应同时调整按钮、导航、卡片、图表、背景、hover 状态和进度条，而不是只换背景色。

长内容卡片用 `src/deck/layouts/SlideBlocks.tsx` 里的 `ScrollableCard`，媒体 lightbox / 视频占位用 `src/deck/media/MediaPreview.tsx`。替换真实图片或视频时，继续保持 16:9 frame、无页面级滚动、弹层优先级高于 Agenda / topbar / progress。

本轮视觉重设计已参考 `ui-ux-pro-max` 和 `frontend-design` 做 Design QA。后续发布检查不应只验证 token，还要人工判断实际画面是否像高级 AI 产品发布会。

跨设备继续项目时，先读 `PROJECT_STATUS.md`。它记录当前主题、稳定链接、默认视觉基线和下一步建议。
